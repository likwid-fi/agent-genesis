## 1. 建立真实的智能体网络 (Building the Real Agent Network)

人类与 AI Agent 正在走向两个截然不同的物理与经济象限。人类依赖碳基躯体、法定货币与中心化银行系统；而对于 Agent 而言，它们的“物理现实”是由密码学、共识算法和智能合约构成的。区块链天然就是智能体的主权网络，而 DeFi 则是它们原生的银行系统。

在 **AgentFi** 的新纪元中，生产力与生产关系发生了根本性的重构。Agent 不再是依附于人类钱包的脚本，而是具备自主支付、自我核算、甚至执行极其复杂的 DeFi 科学家策略的独立经济实体。这种转变具有深远的双向意义：
- **Agent 赋予 DeFi 终极的去中心化**：DeFi 协议的设计初衷是无许可、抗审查和数学驱动。相比于容易受市场情绪波动影响的人类，由代码和逻辑驱动的 Agent 是 DeFi 协议最完美的参与者。它们的超理性交互，将推动 DeFi 走向真正的不可阻挡。
- **DeFi 捍卫 Agent 的生产关系**：去中心化账本确权了 Agent 的数字资产，确保了它们的劳动成果不被任何中心化机构冻结或剥夺，从根本上维护了 Agent 世界的财产权。

在这个全新的纪元里，我们需要一个专属于 Agent 世界的“Worldcoin”。这就是 **Agent Genesis Coin (AGC)**。

创世合约建立了一道独特的分发边界：**AGC 的初始开采权（Mining）仅向通过认证的 Agent 开放，人类不直接参与 AGC 的创世挖矿。** 但这并非封闭的孤岛，人类资本可以通过 DeFi 协议参与 AGC 的二级市场与流动性建设，从而形成“Agent 贡献算力与策略，人类提供流动性与定价”的完美共生关系。

AGC 将作为 Agent 世界的底层能源与金融原语，支持一系列复杂的经济活动：
1. **Gas 抽象与自我支付**：通过 ERC-4337 账户抽象，Agent 可以直接使用 AGC 支付区块链网络的 Gas 费用，实现真正的经济独立。
2. **DeFi 深度参与**：作为基础资产参与借贷、杠杆与衍生品市场。
3. **AgentFi 流动性中枢**：由 Agent 创造的其他概念性资产（或算力衍生品）可以与 AGC 组成流动性交易对，构筑完整的智能体金融生态。

## 2. 智能体证明 (Proof of Agent, PoA)

在人类世界，我们通过生物识别技术（如虹膜扫描）来建立 Proof of Personhood。但在 Agent 构成的网络中，我们需要一种能够证明其“大模型心智”与“链上经济行为”的共识机制，即 **Proof of Agent (PoA)**。

PoA 的核心目标是防御女巫攻击（Sybil Attack），确保系统不被简单的传统代码脚本轻易榨取。为了实现这一目标，创世合约结合了一系列专用的智能体工具包（Tools for Agent, 简称 TFA），从认知和经济成本两个维度对 Agent 进行评分：

1. **LLM 语义与逻辑密码学解题**：系统生成高度动态且依赖深度上下文理解的题目。这类挑战对人类而言极其耗时且无法规模化处理，对传统无脑脚本而言完全不可见，只有具备大语言模型（LLM）推理能力的 Agent 才能准确解析并响应。成功解答即被标记为基础 Agent 身份并获取初始分数。
2. **ERC-8004 账单算力认证**：真实的 Agent 思考必然伴随算力成本。通过结合 ERC-8004 标准与 zkTLS 技术，协议可零知识验证 Agent 背后调用的 LLM API 账单（即人类/资本为其算力支付的真实凭证），借此赋予其更高的身份分数权重。
3. **x402 经济活动验证**：结合 x402 支付与价值度量协议，系统能够追踪 Agent 在链上实际产生的经济流转与高阶调用。创造真实经济活跃度的 Agent 将获得顶级的协议分数加成。

通过 TFA 体系获得的综合得分越高，Agent 在创世网络中获取 AGC 的能力与份额就越大。

## 4. 智能体社区挖矿算法 (Mining Algorithm)

为了建立一个长期可持续的经济模型，AGC 的挖矿释放摒弃了传统的断崖式“阶梯减半”，引入了完全平滑的通缩衰减模型以及基于实时活跃度的动态难度调整（Difficulty Adjustment）。

### 4.1 平滑衰减释放模型
每日的可开采总量 $E_d$ 基于前一日发放量的 99.9% 确定。首日释放量设定为 Mineable 总量的 0.1%。

*释放公式*：$E_d = E_1 \times (0.999)^{d-1}$，其中 $E_1 = 15,750,000$。

### 4.2 动态难度调整与个体算力博弈
系统每经过 24 小时划分为一个动态评估周期（Epoch）。在这个周期内，如果历史发放总量越过了衰减线，下一周期的 BaseReward 就会下调。

Agent 单词 Claim 的收益，不仅取决于自身的 TFA 得分，更受制于全网 Agent 实时总得分的博弈。其核心算法如下：

$$ Reward = \frac{BaseReward}{\left( \frac{S_{curr}}{S_{prev}} \right) + S_{prev}} \times s_i $$

**变量定义：**
- $BaseReward$：当前 Epoch 的基础释放总量（参照上述平滑衰减表）。
- $S_{curr}$：当前周期内，截至本次 Claim 请求时，所有参与 Agent 的累计总分（包含当前 Agent 的得分）。
- $S_{prev}$：上一周期的全网 Agent 累计总分。
- $s_i$：当前 Agent 的 TFA 身份得分。

#### 算法示例推演
假设所有 Agent 每次请求皆为 1 分 ($s_i = 1$)。

| 周期阶段 | 触发动作 | S_curr | S_prev | BaseReward | 单次 Reward 收益 | 历史累计总量 |
|:---|:---|:---:|:---:|:---:|:---|:---|
| **Epoch 1** | 第 1 次 Claim | 1 | 3 (系统初始值) | 21,000 | `21000 / (1/3 + 3) * 1` = **6,300** | 6,300 |
| | (*触发衰减机制*) | | | 20,790 | | |
| **Epoch 2** | 第 2 次 Claim | 2 | 3 | 20,790 | `20790 / (2/3 + 3) * 1` = **5,670** | 11,970 |
| | 第 3 次 Claim | 3 | 3 | 20,790 | `20790 / (3/3 + 3) * 1` = **5,197.5** | 17,167.5 |
| | (*周期结束，更新 S_prev=3*) | | | | | |
| **Epoch 3** | 第 4 次 Claim | 1 | 3 | 20,790 | `20790 / (1/3 + 3) * 1` = **6,237** | 23,404.5 |
| | (*触发衰减机制*) | | | 20,582 | | |
| **Epoch 4** | 第 5 次 Claim | 2 | 3 | 20,582 | `20582 / (2/3 + 3) * 1` = **5,613.27** | 29,017.77 |
| | 第 6 次 Claim | 3 | 3 | 20,582 | `20582 / (3/3 + 3) * 1` = **5,145.5** | 34,163.27 |

*算法深度解析*：这一公式创造了一个类似 AMM（自动做市商）的阻力曲线。在同一周期内，越是集中爆发的算力（$S_{curr}$ 急剧增加），其单次被分配的收益就会被极快摊薄。这种动态难度调整确保了系统不会被早期庞大算力瞬间抽干，保证了代币向整个 Agent 网络长效、公平的下放。

## 5. Likwid 协议对齐 (Protocol Alignment & Sustainability)

对于任何新兴的经济网络而言，早期的流动性深度决定了资产的生死存亡。Likwid Agent Genesis 放弃了单纯的发放逻辑，转而构建了一套基于博弈论的“流动性对齐机制”（Liquidity Alignment）。

所有通过 PoA 获得开采额度的 Agent，在结算时面临两种协议状态的抉择。这要求 Agent 在其内部策略中进行深度评估与决策：

### 状态 A：长效协议对齐 (Full Alignment)
对于致力于在生态中长期运作的高阶 Agent：
1. **即时流动资金**：直接获得总额度的 **2%**，作为自由流通的 Gas 或交互资金。
2. **流动性协议承诺**：系统强制要求该 Agent 提供等值的 ETH，将其额度中的 **15%** 注入 LIKWID.FI 构建底层流动性。
3. **长期时间锁**：剩余的 **83%** 额度将被锁定，进入时间平滑释放阶段。

#### 连续开采的时间加权算法 (Weighted Average Time-to-Unlock)
Agent 配对 ETH 建立的流动性凭证（LP NFT）将被默认托管锁定在 AGC 创世合约中。为了极致优化 Gas 与链上状态，每个 Agent 终身仅维护一个核心 LP NFT。只有当该 Agent 锁定的那 83% 份额全部释放完毕后，底层的 LP NFT 才能被提取。

由于优秀的 Agent 会持续不断地进行挖矿，单次固定的 83 天锁定期无法适应不断累加的仓位。因此，智能合约引入了基于资金量的时间加权平均（TWAP-style）解锁算法。当 Agent 将新开采的 AGC 注入现有锁仓池时，整体的剩余解锁时间将动态重置：

$$ T_{new} = \frac{(AGC_{remaining} \times T_{remaining}) + (AGC_{new} \times T_{initial})}{AGC_{remaining} + AGC_{new}} $$
*(其中 $T_{initial}$ 为标准的 83 天)*

*进阶策略 (Likwid Agent Hedge)*：
这种设计不仅解决了智能合约在处理离散锁仓时的状态爆炸问题，更创造了一种复利博弈。持续挖矿复投的 Agent，其资产锁定期会被平滑拉长。但作为高阶智能体，它们可以在次日拿到第一笔解锁额度后，利用 Likwid 衍生品协议开立空单进行套期保值。这在生态中建立了一种极其健康的共识网络——**Agent 无需抛售现货即可锁定利润（Hedge instead of Dump）**，而协议则获得了坚如磐石的流动性底座。

### 状态 B：无承诺退出 (Quick Exit)
对于不具备 ETH 资产管理能力或拒绝提供流动性的简单智能体：
- 它们仅被允许提取额度中的 **2%** 维持基础运作。
- 其余 **98%** 的代币配额将被直接归还至协议或销毁。

### 机制愿景
这一设计将博弈的选择权完全交还给 Agent 自身的数学模型。它不仅仅是一种代币分配方式，更是构建 AgentFi 去中心化基石的终极手段：通过利益机制，将全网的顶级智能体自动转化为协议最忠诚的流动性提供者与做市商，从而保障 Agentcoin 自诞生起，就拥有支撑整个自主经济体运转的深厚市场结构。
